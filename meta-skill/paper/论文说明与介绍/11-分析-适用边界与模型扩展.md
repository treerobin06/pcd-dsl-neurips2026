## 一、适用边界：DeLLMa 负面结果

### 实验目的

论文不仅展示方法在哪里有效，还诚实展示**在哪里无效**。DeLLMa 实验划定了方法的适用边界。

### DeLLMa 任务描述

- **来源**: Liu et al., 2025
- **任务**: 农业决策——LLM 需要在天气不确定性下选择最优的农业策略
- **核心难点**: 模型必须**从历史上下文中估计概率分布**，而非对给定分布做计算

### 关键区别

| 对比维度 | 本文方法的目标任务 | DeLLMa 任务 |
|---------|-----------------|-----------|
| 分布来源 | 问题中已给定或可推导 | 需要从世界知识/历史数据估计 |
| 瓶颈 | 计算（Compute） | 预测/估计（Predict） |
| DSL 能帮忙吗？ | 能——卸载计算 | 不能——DSL 不做预测 |

### 实验结果

| 模型 | Direct Answer | Compile-time | 随机基线 |
|------|--------------|-------------|---------|
| GPT-4o-mini | 40.0% | 0.0%（代码执行失败） | 29.1% |
| GPT-5.4 | 40.0% | 29.4% | 29.1% |
| Opus 4.6 | 40.0% | 17.6% | 29.1% |

- **数据规模**: 20 个问题（17 个可评估）
- Compile-time 求解器在 GPT-4o-mini 上完全失败
- GPT-5.4 的 compile-time 仅达到随机基线水平

### 论文的明确声明

> "Our DSL excels when distributions are given or derivable from the problem statement; it adds nothing when the bottleneck is prediction rather than computation."

**方法有效的条件**: 概率分布已知或可从问题描述中推导
**方法无效的条件**: 需要模型自己估计/预测概率分布

## 二、模型扩展分析

### 实验目的

回答：**用更强（更贵）的模型能否弥合计算差距？**

### 偏好学习上的模型梯度

| 模型 | Compute\|Gold | 相对 GPT-4o-mini 价格 |
|------|--------------|---------------------|
| GPT-4o-mini | 28% | 1× |
| GPT-4o | 30% | ~10× |
| GPT-5.4 | 40% | ~67× |
| Sonnet 4 | 64% | ~20× |
| Gemini 3.1 | 69% | ~30× |
| Opus 4.6 | 78% | ~100× |
| **DSL** | **100%** | **~0.001×** |

### BN 推断上的深度退化

| 深度 | GPT-4o-mini | GPT-5.4 | 提升 |
|------|-------------|---------|------|
| 2 | 82% | 81% | -1pp |
| 5 | 11% | 24% | +13pp |
| 10 | 3% | 11% | +8pp |
| **整体** | **22.3%** | **31.2%** | **+8.9pp** |

### 关键发现

1. **更强的模型确实能提高 Compute 准确率**：Opus 4.6 (78%) > GPT-4o-mini (28%)
2. **但提升远远不够**：
   - 即使 Opus（最强模型），仍有 22% 的计算错误率
   - GPT-5.4 在 BN 深度 10 上仍有 89% 的错误率
   - 67× 的价格提升只换来 9pp 的准确率提升
3. **PAL + GPT-5.4 也不完美**: 98.1% 但仍不是 100%，且在大网络上崩溃
4. **没有任何模型-工具组合能保证正确性**（DSL 除外）

### bnlearn 上的彻底失败

在 ≥20 节点的真实网络上：
- Compute|GoldParse: **两个模型都是 0%**
- PAL: **0-3%**
- DSL: **100%**

模型扩展在这里**完全失效**——不是差几个百分点的问题，而是根本不 work。

## 三、综合适用性图谱

```
                    分布已知/可推导          需要估计分布
                    ┌──────────────┐     ┌──────────────┐
                    │              │     │              │
小规模计算          │  DSL 有效     │     │  DSL 无效     │
(浅网络/少变量)     │  PAL 也还行   │     │  (DeLLMa)    │
                    │              │     │              │
                    ├──────────────┤     ├──────────────┤
                    │              │     │              │
大规模计算          │  DSL 有效     │     │  DSL 无效     │
(深网络/多值变量)   │  PAL 完全失败  │     │  (未测试)     │
                    │              │     │              │
                    └──────────────┘     └──────────────┘
```

DSL 方法的有效区域：**左列**（分布已知的计算任务）
DSL 方法的无效区域：**右列**（需要从上下文估计分布的任务）

## 对审计者的提示

在审计这部分内容时，值得注意以下几点：

1. **DeLLMa 实验规模很小**（仅 20 个问题，17 个可评估），统计力度有限
2. **论文没有测试"大规模 + 需要估计分布"的象限**
3. **Opus 的 78% Compute 准确率是否意味着再强一些的模型就能接近 100%？** 论文暗示不太可能（因为 BN 深度上的退化模式很一致），但没有给出理论上限
4. **DSL 在连续分布任务上的表现未知**——论文明确声明 DSL 仅覆盖离散精确推断
